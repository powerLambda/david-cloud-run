## Plan: CalDAV to ICS Cloud Run Service

Build a new Go service in the empty workspace that exposes a public HTTP endpoint returning an ICS feed from Feishu CalDAV, filtered to a 2-month window (past + future), with recurrence expansion and Asia/Shanghai timezone, and deployable to Cloud Run.

**Steps**
1. Establish the Go module using the provided module path, add a minimal HTTP server that binds to the Cloud Run port environment variable, and define configuration via environment variables for CalDAV URL, username, and password. Add a simple health endpoint for readiness checks.
2. Implement a CalDAV client that authenticates to the Feishu CalDAV server, discovers the principal, and selects the default calendar (first calendar in the list). Fetch events with a time range from now minus one month to now plus one month, using Asia/Shanghai as the reference timezone.
3. Convert CalDAV results to an ICS calendar response, following the reference Python logic of copying event subcomponents into a new calendar and setting standard prodid/version metadata. Ensure the output is served as text/calendar with an inline filename and consistent line endings.
4. Expand recurring events into instances within the 2-month window, using either CalDAV server-side expand support or a Go recurrence library if the server does not expand results by default.
5. Implement timeouts and error handling for CalDAV fetch failures (return 5xx with a clear message).
6. Wire the public endpoint at the requested path so it returns the ICS content, and add basic logging for request start/finish and CalDAV latency.
7. Add container build configuration and a short deployment guide for Cloud Run, including required environment variables and the webcal subscription URL format.

**Relevant files**
- None yet; the workspace is empty, so new module and configuration files will be created during implementation.

**Verification**
1. Run the service locally with env vars set and call the endpoint to confirm it returns a valid ICS with events in the 2-month window and expanded recurrences.
2. Confirm timezone handling by checking at least one event’s DTSTART/DTEND offsets against Asia/Shanghai.
3. Confirm the known existing CalDAV event appears in the ICS output within the window.
4. Deploy to Cloud Run and verify a webcal subscription in Google Calendar updates correctly.

**Decisions**
- Public endpoint with no auth.
- CalDAV credentials via environment variables.
- Module path: github.com/powerLambda/david-cloud-run.
- Endpoint path: /caldav2ics/feishu.
- Calendar selection: default calendar.
- Timezone: Asia/Shanghai.
- Time range: past one month plus upcoming one month.
- Recurrences expanded into instances in the window.

**Further Considerations**
1. CalDAV server specifics: if Feishu requires a non-default calendar or special URL path, we may need an explicit calendar identifier env var.
